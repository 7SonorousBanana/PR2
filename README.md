# Міні-сервіс прогнозування погоди
# Опис проєкту

Даний проєкт реалізує прототип міні-сервісу прогнозування опадів на основі історичних метеорологічних даних.  
Сервіс отримує дані з відкритого API Open-Meteo, навчає модель машинного навчання та дозволяє зробити прогноз опадів через простий веб-інтерфейс, створений за допомогою Streamlit.
Мета проєкту - продемонструвати повний цикл роботи з даними:

- отримання даних через HTTP API
- підготовка даних
- навчання ML-моделі
- оцінка якості моделі
- прогнозування
- візуалізація результатів у веб-застосунку

# Дані, які використовуються

Дані отримуються з Open-Meteo API  
https://open-meteo.com/

Використовуються щоденні метеодані.

Основні змінні:

- `precipitation_sum` - загальна кількість опадів за день
- `rain_sum` — кількість дощу за день
- `temperature_2m_max` - максимальна температура
- `temperature_2m_min` - мінімальна температура
- `temperature_2m_mean` - середня температура
- `wind_speed_10m_max` - максимальна швидкість вітру
- `wind_gusts_10m_max` - максимальні пориви вітру

Дані завантажуються за заданими координатами (за замовчуванням використовується Київ) і за вказаний часовий період.
Отримані дані зберігаються у файл: weather_daily.csv

# Цільова змінна

Для задачі класифікації використовується змінна наявності опадів.
Вона створюється на основі `precipitation_sum`.

0 — опадів немає
1 — опади є

Правило формування:

precipitation_sum == 0 → 0
precipitation_sum > 0 → 1

# Використані моделі машинного навчання

Random Forest
Ансамблева модель на основі дерев рішень.  
Вона здатна знаходити складні залежності між ознаками і зазвичай дає кращу точність.

# Метрики оцінки моделі

Якість моделі оцінюється за такими метриками:

- Accuracy - загальна точність моделі
- Precision - точність прогнозів позитивного класу
- Recall - повнота виявлення позитивного класу
- F1-score - баланс між precision і recall
- Confusion Matrix - матриця помилок класифікації

Також виводиться Classification Report, який містить всі основні метрики для кожного класу.

# Структура та організація проєкту

Проєкт має наступну структуру:
app.py - Основний файл застосунку.
requirements.txt - Файл зі списком бібліотек Python, необхідних для запуску проєкту. (streamlit, pandas, numpy, requests, scikit-learn)
weather_daily.csv - CSV файл з історичними погодними даними, отриманими з Open-Meteo.

# Послідовність роботи програми
1. Отримання даних
Користувач задає:

- координати (опціонально)
- початкову дату
- кінцеву дату

Після натискання кнопки Отримати дані з Open-Meteo програма:

1. робить HTTP-запит до API
2. отримує погодні дані
3. формує таблицю
4. зберігає її у CSV

2. Підготовка даних
- створюється цільова змінна
- додаються календарні ознаки
- беруться дані попередніх днів

3. Навчання моделі
Дані діляться на:

80% — навчання
20% — тест

Після цього модель:

- навчається на train-даних
- перевіряється на test-даних
- обчислюються метрики точності

4. Прогнозування

Користувач натискає кнопку Зробити прогноз.
Модель аналізує ознаки дня і видає результат:
"Очікуються опади" або "Опадів не очікується"
